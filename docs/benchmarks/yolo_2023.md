# yolo_2023 (extended track)

TinyYOLO objectness verification: 72 instances, one network (`TinyYOLO.onnx`,
ResNet-style ReLU CNN — 14 Conv, 7 ReLU layers, skip-Adds, identity Pads,
2×2/2 AveragePools), eps = 1/255, 300 s per-instance budgets. Every spec is a
single CONJUNCTIVE disjunct: an AND of 5 objectness constraints `Y_i ≤ −1`;
refutation closes when ANY one conjunct is refuted (`lo(Y_i) + 1 > 0`). No SAT
cases exist in the instance set.

## What it took (VC had never run this benchmark)

1. **Identity-Pad soundness** (`onnx_optimizer.drop_identity_pads`, ungated in
   `ComputeGraph.optimize()`): TinyYOLO carries `Pad(pads=[0]*8)` no-ops.
   Also fixed a latent UNSOUND loader bug — attribute-style (opset ≤ 10) Pad
   params were silently ignored (`params={}`), so `PadNode` passed through
   even for non-zero pads. The loader now parses `attrs['pads']/['value']`,
   and `PadNode` raises on missing/unsupported pads. Pinned by
   `tests/test_drop_identity_pads.py`; point-prop vs onnxruntime err 1.9e-6.

2. **AveragePool as equivalent conv** (gpu_graph serializer in `network.py`):
   `AveragePool(k, s, pad=0)` ≡ Conv with a depthwise-uniform C×C×kH×kW
   kernel (`1/(kH·kW)` on the diagonal). Emitting type `'conv'` makes every
   downstream chain (zono forward, CROWN backward, gen-LP, MILP, PGD) support
   it with zero new handlers. Non-zero padding refuses loudly
   (`count_include_pad` semantics not expressible as a plain conv).

3. **Conjunctive-disjunct ANY-closure** (completeness fix, commit f303be8):
   both `verified_disj` predicates in `verify_graph.py` required
   `all(spec_lbs > 0)` over a disjunct's conjuncts — sound but yolo was
   unverifiable BY CONSTRUCTION (e.g. `prop_000299` has a conjunct at
   lo ≈ −8, individually unrefutable, while plain zono already refutes
   sibling `Y_760` with margin +0.49). Fixed to `any(...)` at both sites,
   plus phase-2.5 disjunct closure and break-on-close in the per-query loop
   (saves ~26 s per sibling query). For single-conjunct disjuncts (all
   regular-track benchmarks) `any() == all()` — behavior-neutral, cora spot
   checks unchanged.

4. **Sibling-halfspace dual cuts** (INVPROP-lite, `phase8_sibling_halfspaces`,
   default on): for a conjunctive disjunct, refutation may assume the SAT
   set, so every sibling conjunct `w_j·y + b_j ≤ 0` is projected through the
   output zonotope map into a generator-space halfspace `a·e ≤ b` and
   dualized (ν ≥ 0) into every Phase-8 BnB node bound
   (`fast_verify_topk.node_bound_logbucket`). Sound by weak duality for any
   ν ≥ 0; bit-exact no-op when no siblings exist (M = 0), so all
   regular-track benchmarks are untouched. Measured A/B on yolo
   (`scratch/yolo_sweep_2026-06-09/yolo_ab_hs.log`): parity walls on solved
   cases (37.1 s vs 36.9 s; 167.1 s vs 167.3 s), but NON-BINDING in practice —
   it neither cracks `prop_000024` (still timeout) nor speeds `prop_000308`
   (296.93 s vs 296.9 s). Kept on (free + pinned by
   `tests/test_sibling_halfspaces.py`); the 62/72 score stands without it.

## Config (`configs/yolo_2023.yaml`)

Lean tinyimagenet-style all-GPU profile: `max_tighten_layer: 0`,
`phase2_crown_enabled: false`, `phase8_use_dual_ascent_gpu: true`,
`phase8_min_budget_frac: 0.5`, plus `zono_lift_enabled: true` — the per-query
α-CROWN pass closes conjuncts that plain zono/joint-α can't (`prop_000298`:
best conjunct −3.3 at zono, closed by the per-query pass). The default
interleaved MILP-tighten cascade burned 163 s of the 300 s budget on the
first probe — keep it off. Do NOT blindly raise
`bab_refine_phase05_spec_iters` (200/100 diverged to −inf on the probe).

## Known unsolved (10)

`prop_000024` (ABC solves in 34.2 s — β-CROWN BaB with INVPROP-refined
intermediates; our Phase-8 BnB times out) and 9 both-timeout cases. ABC's
speed edge (8.5 s flat on easy cases vs our 28–40 s) comes from INVPROP
output-constraint intermediate refinement (`apply_output_constraints_to:
["/input-48"]` in their `vnncomp24/yolo-xiangruzh.yaml`) — backward-tightening
intermediate bounds under the assumed output constraints. We only exploit the
output constraints at the BnB dual (sibling halfspaces); an INVPROP-style
intermediate-bound analog is the known lever for both the misses and the
speed gap (also relevant to cifar100 / metaroom q8).
