### Soundness note — reverse-g false-verify (FIXED 2026-06-07)

The first lean sweep (with `phase8_reverse_g:true`, ported from tinyimagenet) hit
**6 FALSE_VERIFY, all on `CIFAR100_resnet_large`, 0 on medium**. Root cause was
**NOT a backward-math bug** (`build_state_reverse` is bit-exact vs autograd on
conv/strided-conv/fc/relu/add-skip — see `tests/test_reverse_g.py`). It was a
**formulation-tag dispatch bug**: `build_state_reverse` (and the batched
`reverse_batched.py`) returned `formulation='sparse'` while their per-neuron data
is in the α-zono parametrization (`y=λz+μ(1+e_new)`). `build_gen_lp_from_state`
dispatches the MILP builder on that string; `'sparse'` routed α-zono data into the
generic `y=e_new∈[0,hi]` direct-ReLU builder — a coordinate-system mismatch that
built a too-tight (UNSOUND) polytope. The **high-bin MILP fallback** then read
min-margin **+0.235** (vs the correct **−0.146**) and declared the query CLOSED →
false `unsat`. The dual-ascent BnB reads the state fields directly (ignores the
tag) so it stayed sound — which is why the bug surfaced *only* through the MILP
fallback, and *only* on the reverse path (the fast reverse build left budget to
reach the fallback; the forward build timed out first).

**Fix:** tag the reverse state `formulation='alpha_zono'` (one line in each of
`reverse_g.py` and `reverse_batched.py`). Re-validated: all 15 resnet_large SAT
cases return timeout/not-unsat with sat-off (0 false-verify). `phase8_reverse_g`
is **re-enabled** (it is ~3.3× faster at state build: 2.63s→0.80s on prop_1063).
The integration test pins prop_6583 (resnet_large) as a `sat` soundness guard.
See memory `project_reverse_g_large_resnet_unsound` and CLAUDE.md's `entry['form']`
dispatch contract.

Note: on cifar100 the high-bin MILP fallback closes **zero** queries — the
dual-ascent BnB carries every verified case (measured: 6/6 resnet_medium UNSAT
closed by BnB, fallback fired 0×). The fallback was pure soundness surface here.

## Knobs (`configs/cifar100_2024.yaml`)

**Lean overrides (2026-06-07), on top of the base config documented below:**
- `max_tighten_layer: 0` — skip the Phase-1 Gurobi MILP-tighten cascade (~13s/case,
  CPU). α-CROWN (phase 0.5) closes nearly all disjuncts; the fast GPU Phase-8 BaB
  closes the rest from the phase-0.5 bounds. This is the main 2–4× UNSAT speedup.
- `phase8_reverse_g: true` — reverse-mode α-zono state build (~3.3× faster on
  resnet_large); sound since the 2026-06-07 formulation-tag fix (see soundness
  note above). `phase8_reverse_g_batched: false` (batched path fixed too but not
  yet enabled here).
- `parallel_pgd_enabled: false` — the in-MILP-window PGD attacker is dead with no
  MILP windows (waits on a lock that never sets).
- `phase26_pre_cascade_total_frac: 0.15`, `phase26_pre_cascade_per_spec_cap: 15.0`
  — pre-cascade PGD KEPT (and budget bumped from 0.10/5.0) as the SAT catcher: it
  recovers the 5 resnet_large SAT cases the old config caught via parallel-PGD
  during the MILP windows (prop_3585 needs ~13s of attack). UNSAT cost ~+5-12s
  (cases still ~34s, < budget).

Base-config rationale (each non-default override exists for a measured reason):

- `phase8_use_dual_ascent_gpu: true` — GPU dual-ascent BaB in Phase 8; required to fit prop_6232-class cases in the 10GB GPU budget without OOM (commit 466a370).
- `phase8_dual_ascent_max_iter: 1` — K=1 per node. Higher gives marginal tightness for cubic extra cost.
- `zono_lift_enabled: false` — Phase 2.5 zono-lift adds 3-5s; superseded by α-CROWN Phase 0.5 + Phase 1 cascade.
- `phase2_crown_enabled: false` — Phase 2 CROWN spec backward is redundant; Phase 0.5 batched α-CROWN already produced tighter `spec_lbs`.
- `phase8_min_budget_frac: 0.5` — guarantee Phase 8 gets ≥ 50s of the 100s budget (trims overruns from earlier phases).
- `pgd_middle_enabled: false` — Phase 3.5 PGD redundant; Phase 2.6 per-spec PGD is strictly better.
- `phase26_pgd_per_spec_min_per_spec: 0.5` — per-spec PGD budget bumped from 0.2s (caught 2 missed SATs on the v4 sweep).
- `phase26_pgd_per_spec_strict_min: true` — honor the per-spec min even if total budget tight.
- `phase26_pre_cascade_total_frac: 0.10` + `phase26_pre_cascade_per_spec_cap: 5.0` — pre-α-CROWN PGD hook (zono-sorted) gets ~10% of remaining wall, capped 5s/spec. Catches case-22-class SAT misses where Phase 2.6 PGD's smaller budget would have missed.
- `parallel_pgd_enabled: true` — worker thread runs PGD on the open-after-Phase-0.5 disjuncts during the cascade's MILP windows (~13s of pure-CPU MILP per case). α-CROWN unaffected (validated empirically). Net win: ~10s on UNSAT cases.
