## Knobs (`configs/cifar100_2024.yaml`)

Each non-default override exists for a measured reason:

- `phase8_use_dual_ascent_gpu: true` — GPU dual-ascent BaB in Phase 8; required to fit prop_6232-class cases in the 10GB GPU budget without OOM (commit 466a370).
- `phase8_dual_ascent_max_iter: 1` — K=1 per node. Higher gives marginal tightness for cubic extra cost.
- `zono_lift_enabled: false` — Phase 2.5 zono-lift adds 3-5s; superseded by α-CROWN Phase 0.5 + Phase 1 cascade.
- `phase2_crown_enabled: false` — Phase 2 CROWN spec backward is redundant; Phase 0.5 batched α-CROWN already produced tighter `spec_lbs`.
- `phase8_min_budget_frac: 0.5` — guarantee Phase 8 gets ≥ 50s of the 100s budget (trims overruns from earlier phases).
- `pgd_middle_enabled: false` — Phase 3.5 PGD redundant; Phase 2.6 per-spec PGD is strictly better.
- `phase26_pgd_per_spec_min_per_spec: 0.5` — per-spec PGD budget bumped from 0.2s (caught 2 missed SATs on the v4 sweep).
- `phase26_pgd_per_spec_strict_min: true` — honor the per-spec min even if total budget tight.
- `phase26_pre_cascade_total_frac: 0.10` + `phase26_pre_cascade_per_spec_cap: 5.0` — pre-α-CROWN PGD hook (zono-sorted) gets ~10% of remaining wall, capped 5s/spec. Catches case-22-class SAT misses where Phase 2.6 PGD's smaller budget would have missed.
- `parallel_pgd_enabled: true` — worker thread runs PGD on the open-after-Phase-0.5 disjuncts during the cascade's MILP windows (~13s of pure-CPU MILP per case). α-CROWN unaffected (validated empirically). Net win: ~10s on UNSAT cases.

## Reproducing a single case

```bash
ssh stan@100.83.144.97
cd ~/Desktop/temp/vibecheck-temp
./.venv/bin/python /home/stan/persistent_runs/scripts/runner_v8.py \
  ~/repositories/vnncomp2025_benchmarks/benchmarks/cifar100_2024/onnx/CIFAR100_resnet_large.onnx \
  ~/repositories/vnncomp2025_benchmarks/benchmarks/cifar100_2024/vnnlib/CIFAR100_resnet_large_prop_idx_6583_sidx_8860_eps_0.0039.vnnlib \
  100
```

`runner_v8.py` is the v9 sweep runner (parallel PGD enabled). For YAML-driven runs use `main.py --config configs/cifar100_2024.yaml`.

## Reproducing the full sweep

```bash
ssh stan@100.83.144.97 'tmux new-session -d -s sweep "SWEEP_CATEGORIES=cifar100_2024 SWEEP_OUT_DIR=/home/stan/persistent_runs/results/sxs_$(date +%Y%m%d) \
  ~/Desktop/temp/vibecheck-temp/.venv/bin/python /home/stan/persistent_runs/scripts/sweep_sxs.py 2>&1 | tee /home/stan/persistent_runs/results/sweep_$(date +%Y%m%d).log"'
```

Sweep wall ≈ 3-4 hours on a fresh GPU. Results materialize as JSONL per case + abc; per-case stdout flagged AB-WIN/VC-WIN/DISAGREE.
