# cora_2024 — vibecheck benchmark record

VNNCOMP 2025 regular track. CORA verifier benchmark: 9 MLPs trained
on mnist/cifar10/svhn × {point, set, trades} regimes. 180 instances.

All 9 models share the same architecture: 8 FC layers (250-wide
hidden, 10 output) with 7 ReLUs. The cifar10/svhn `set` and `trades`
variants prepend an input-normalization layer (scalar Mul + scalar Add).
Specs are multi-disjunct DNF (9 disjuncts: `y_target >= y_j` for each
non-target class).

## Final score (server1, 2026-05-24, RTX 3080 / 10 GB, 30 s budget)

| Solver | Solved / 180 | Wall (total) | Notes |
| --- | --- | --- | --- |
| **vibecheck** | **153** | **~15 min** | 131 SAT + 22 UNSAT |
| AB-CROWN (published, 2025) | 153 | ~34 min | 131 SAT + 22 UNSAT, 27 timeouts |

**Tied verdict-wise, 2.3× faster wall.** All 27 unknowns are cases
AB-CROWN also times out.

Per-case: SAT cases sub-second (PGD root); UNSAT cases close via
MILP at L1+L2 → per-neuron adaptive backward CROWN at L3-L6 →
Phase 8 dual-ascent BaB (~14 s on the hardest case).

The previously-stuck case `cifar10-set img100` (where AB-CROWN uses
β-CROWN BaB) is now verified in ~28 s using MILP tightening at L1+L2
followed by per-neuron adaptive backward CROWN to propagate the
tightening downstream to L3-L6 (~30 ms overhead), then Phase 8
dual-ascent BaB.

We win on wall time: ~2× faster total across the suite (no 6-7 s
per-case overhead — AB-CROWN re-warms α for every instance).

## Algorithmic adds for this benchmark

This benchmark exposed three latent bugs in the cross-model dispatch
that surfaced here for the first time:

- **`n_output` was wrong for Conv-output models** (`_phase1_bab_refine`
  in `verify_graph.py`). The serialized op-list passed to BaB strips
  shape metadata for `add` ops; the old fallback used the last ReLU's
  width as `n_output`, which is the hidden layer (250) not the output
  Add's width (10) on cora's `*-set`/`*-trades` models. Now uses the
  live `z_final_initial.center.numel()` as the source of truth.
- **Scalar-bias broadcast in CROWN backward**: cifar10/svhn `*-set` and
  `*-trades` models prepend `Mul(scalar) + Add(scalar)` for input
  normalization. The `Add` backward did `ew @ bt` with `bt = bias.flatten()`
  shape (1,) and `ew` shape (3072,) → matmul size mismatch. New helper
  `_bias_dot_ew` in `alpha_crown.py` handles `numel == 1` via
  `ew.sum(dim=-1) * scalar`. Applied at every bias-matmul site in
  `alpha_crown.py` and `verify_graph._adaptive_spec_lb`.
- **`mul` op handler missing** in 4 backward/forward passes:
  `_forward_zonotope_graph`, `_spec_backward_graph`, `alpha_crown.py`
  (3 dispatch loops), `verify_gen_lp.py` (forward + 2 precompute loops).
  Added via the shared `_mul_scale_backward` / `_mul_scale_zono` helpers.

## Knobs (`configs/cora_2024.yaml`)

Borrows the cifar100-style Phase 8 dual-ascent config: turn off
`zono_lift_enabled` / `phase2_crown_enabled` / `pgd_middle_enabled`
(they don't help on cora's 250-wide MLPs and would steal budget),
shrink `bab_refine` to one pass, and give Phase 8 at least 70 % of
total budget (`phase8_min_budget_frac: 0.7`). Without this config
we lose a 2nd UNSAT case (cifar10-set img96).

## Reproducing a single case

```bash
.venv/bin/vibecheck \
  --net path/to/cora_2024/onnx/mnist-set.onnx \
  --spec path/to/cora_2024/vnnlib/mnist-img7.vnnlib \
  --mode graph --timeout 30 --bits 32 \
  --config configs/cora_2024.yaml
```
