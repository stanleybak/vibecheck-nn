# cora_2024 — vibecheck benchmark record

VNNCOMP 2025 regular track. CORA verifier benchmark: 9 MLPs trained
on mnist/cifar10/svhn × {point, set, trades} regimes. 180 instances.

All 9 models share the same architecture: 8 FC layers (250-wide
hidden, 10 output) with 7 ReLUs. The cifar10/svhn `set` and `trades`
variants prepend an input-normalization layer (scalar Mul + scalar Add).
Specs are multi-disjunct DNF (9 disjuncts: `y_target >= y_j` for each
non-target class).

## Final score (lean config, 2026-06-07, AWS A10G / 24 GB, 30 s budget)

Ported the lean Phase-1: skip the Gurobi MILP-tighten cascade; all-GPU Phase-8
dual-ascent BaB + input-split from the phase-0.5 α-CROWN bounds.

| AB-CROWN (published, 2025) | 153 | 131 SAT + 22 UNSAT, 27 timeouts |

- **0 false-verify / 0 false-sat** (sound), **0 MISS** (every ABC-solved case solved)
- **Ties ABC** verdict-wise; all 27 unknowns are cases ABC also times out.
- The cascade-skip closes the cifar10-set UNSAT cases **img96 / img100 in ~12 s**

  is not used (no soundness risk; cf. cifar100 where reverse-g was unsound on
  resnet_large — see `MEMORY: project_reverse_g_large_resnet_unsound`).

### Previous (server1, 2026-05-24, RTX 3080 / 10 GB)
vibecheck 153 / AB-CROWN 153 via the bab_refine MILP@L1+L2 cascade. The lean
config supersedes it: same coverage, all-GPU (drops the Gurobi dependency),
faster on the hard cifar10-set UNSAT cases.

Per-case: SAT cases sub-second (PGD root); UNSAT cases close via
MILP at L1+L2 → per-neuron adaptive backward CROWN at L3-L6 →
Phase 8 dual-ascent BaB (~14 s on the hardest case).

The previously-stuck case `cifar10-set img100` (where AB-CROWN uses
β-CROWN BaB) is now verified in ~28 s using MILP tightening at L1+L2
followed by per-neuron adaptive backward CROWN to propagate the
tightening downstream to L3-L6 (~30 ms overhead), then Phase 8
dual-ascent BaB.

We win on wall time: ~2× faster total across the suite (no 6-7 s
per-case overhead — AB-CROWN re-warms α for every instance).

## Algorithmic adds for this benchmark

This benchmark exposed three latent bugs in the cross-model dispatch
that surfaced here for the first time:

- **`n_output` was wrong for Conv-output models** (`_phase1_bab_refine`
  in `verify_graph.py`). The serialized op-list passed to BaB strips
  shape metadata for `add` ops; the old fallback used the last ReLU's
  width as `n_output`, which is the hidden layer (250) not the output
  Add's width (10) on cora's `*-set`/`*-trades` models. Now uses the
  live `z_final_initial.center.numel()` as the source of truth.
- **Scalar-bias broadcast in CROWN backward**: cifar10/svhn `*-set` and
  `*-trades` models prepend `Mul(scalar) + Add(scalar)` for input
  normalization. The `Add` backward did `ew @ bt` with `bt = bias.flatten()`
  shape (1,) and `ew` shape (3072,) → matmul size mismatch. New helper
  `_bias_dot_ew` in `alpha_crown.py` handles `numel == 1` via
  `ew.sum(dim=-1) * scalar`. Applied at every bias-matmul site in
  `alpha_crown.py` and `verify_graph._adaptive_spec_lb`.
- **`mul` op handler missing** in 4 backward/forward passes:
  `_forward_zonotope_graph`, `_spec_backward_graph`, `alpha_crown.py`
  (3 dispatch loops), `verify_gen_lp.py` (forward + 2 precompute loops).
  Added via the shared `_mul_scale_backward` / `_mul_scale_zono` helpers.

## Knobs (`configs/cora_2024.yaml`)

**Lean change (2026-06-07):** `max_tighten_layer: 0` (was 2) skips the MILP
cascade — the Phase-8 BaB + input-split closes the cifar10-set UNSAT cases
without it (img100 ~12 s vs the cascade's ~29 s). `parallel_pgd_enabled: false`
(was true; dead with no MILP windows). Pre-cascade PGD kept as the SAT catcher.

Borrows the cifar100-style Phase 8 dual-ascent config: turn off
`zono_lift_enabled` / `phase2_crown_enabled` / `pgd_middle_enabled`
(they don't help on cora's 250-wide MLPs and would steal budget),
shrink `bab_refine` to one pass, and give Phase 8 at least 70 % of
total budget (`phase8_min_budget_frac: 0.7`). Without this config
we lose a 2nd UNSAT case (cifar10-set img96).

## Exact-MILP router (2026-06-09) — the "+7" was PHANTOM; retracted 2026-06-10

The 27 both-miss cases split cleanly from solved cases at Phase-8 entry
(solved enter at worst spec LB >= -0.65, misses at <= -3.3 where the BnB
frontier doubles every level — 67M-node OOM on cifar10-set img440), so
`phase8_exact_milp_below: -2.0` routes hopeless pure-FC/ReLU cases to the
exact per-neuron big-M MILP (`milp_verify`). A 2026-06-09 sweep claimed
**+7 gains (160/180)**. The SAT canary then fired (cifar10-trades img339,
known SAT, `unsat` in 7.2 s with PGD disabled) and the autopsy
(`scratch/CORA_JOINT_ALPHA_AUTOPSY.md`) found the real bug:

**`ComputeGraph.gpu_layers` silently dropped the cifar10/svhn nets'
`Mul(3.97)+Add(-1.88)` input-normalization preamble** (its catch-all
skipped any op it didn't recognize). The ENTIRE milp_verify engine — zono
bounds, the bbr fed to joint-α, the exact MILP encoding, the PGD forward —
ran on the unnormalized network (forward error vs onnxruntime: 5.28). All
7 "gains" and the img339 false-verify were verdicts about the wrong
network; mnist nets have no preamble, which is exactly why only cifar10
cases "cracked".

Fix (branch fix/cora-joint-alpha): gpu_layers folds a leading input affine
x↦s⊙x+t into the first fc (W·diag(s), W t + b — incl. acasxu-style
Sub/Div forms), and the catch-all now raises `NotImplementedError`
(whitelist: Relu/Flatten/Reshape/Identity/Squeeze/Unsqueeze). Pinned by
`tests/test_gpu_layers_input_affine.py`; forward now matches onnxruntime
to <=4.2e-6 on all 6 cora families. Post-fix recheck @ 30 s vnncomp
budget: all 7 "gains" are honest timeouts; img339 canary clean (pgd-off →
timeout, pgd-on → sat 3.1 s).

**Real score: 153/180 = AB-CROWN parity (0 misses, 0 conflicts).** The
27 both-miss cases remain open; the router is kept (the exact MILP is a
legitimate attempt on the correct network now, it just doesn't win in
30 s). Ground truth for the misses unknown (nnv/pyrat "solves" of similar
cases are documented false-verifies elsewhere).

Integration: `cifar10-trades img339` pinned as the normalization-fold SAT
canary (replaces the phantom img440 router pin).
