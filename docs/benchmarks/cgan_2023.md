# cgan_2023 — vibecheck benchmark record

VNNCOMP 2025 regular track. Conditional-GAN generators: a 1D condition
vector → upsampled / ConvTranspose'd image. The verification target is a
scalar discriminator score; properties check that the output is bounded
within a range for any condition in `[c0 - ε, c0 + ε]`.

21 instances across 7 sub-models:
- **Base cGAN** (8 cases): `imgSz{32,64} × nCh{1,3} × prop{0..3}`. Pure
  ConvTranspose + ReLU.
- **nonlinear_activations** (1 case): `imgSz32_nCh_3` with Tanh and
  Sigmoid inserted between ConvTransposes.
- **transposedConvPadding_1** (1 case): `imgSz32_nCh_1` with
  `ConvTranspose(output_padding=1)`.
- **upsample** (1 case): `imgSz32_nCh_3` with nearest-neighbour
  Upsample-9 instead of ConvTranspose.
- **small_transformer** (2 cases): bilinear-MatMul attention +
  Softmax + max-pool. Our gpu_graph can't represent attention; both SAT
  cases route through the raw-ONNX PGD fallback
  (`onnx_torch_runner.pgd_via_onnx`), witness validated against the
  original ONNX.

  **Load-time dependency: onnxsim.** This model only loads after onnxsim
  constant-folds (a) the spectral-norm weight divisions
  (`weight_orig / sigma`, computed at inference from constants) and (b)
  the dynamic `Shape → Gather → Cast → Concat` reshape subgraphs into
  static form. `onnx_loader.load_onnx` auto-triggers onnxsim when it
  sees `Softmax` + >4 `MatMul`. **onnxsim is a hard dependency**
  (`pyproject.toml`); without it the load crashes on the non-constant
  Gemm weight (`KeyError: onnx::Gemm_246`). The loader also folds
  const/const `Div` and resolves Conv/Gemm/ConvTranspose weights via
  `_const` (initializer *or* folded constant), raising
  `NotImplementedError` on a genuinely dynamic weight rather than a bare
  `KeyError` — so a spectral-norm net without the Softmax trigger still
  loads. Covered by `tests/test_onnx_ops.py::test_div_folded_*` /
  `test_dynamic_*_raises`.

  > **AWS-sweep miss (root cause).** The full VNNCOMP-2025 regular-track
  > sweep reported both small_transformer cases as misses. The cause was
  > the AWS venv lacking onnxsim (an optional dep at the time), so the
  > load crashed before the attention PGD fallback could run. Making
  > onnxsim a hard dependency closes this permanently; both cases now
  > return `sat` in ~11 s locally, matching AB-CROWN (~8.4 s).

## Final score (server1, 2026-05-23, RTX 3080 / 10 GB, 120 s budget)

| Solver | Solved / 21 | Wall (total) | Notes |
| --- | --- | --- | --- |
| **vibecheck (120 s/case)** | **21** | **~55 s** | 11 SAT + 10 UNSAT |
| AB-CROWN (published, 2025) | 21 | ~170 s | 6-15 s/case |

Per-case dominance: imgSz32 UNSAT bowls **4.6 s** vs AB-CROWN 8.7 s (2× faster);
imgSz64 UNSAT 7.1 s vs 12 s; SAT sub-second across the board.
The single case AB-CROWN beats is imgSz64 nCh_3 prop_1 (7 s ours vs 15 s
theirs — both well inside budget).

**Re-verified 2026-06-08 (local, with onnxsim hard-dep):** full 21-case
sweep with `configs/cgan_2023.yaml`, all 21 verdicts match AB-CROWN
(11 SAT + 10 UNSAT). The 2 small_transformer SAT cases — missed in the
earlier AWS sweep due to a missing onnxsim — now solve in ~11 s each.

### Bottleneck history

Initial implementation took **14 s/UNSAT** (matching AB-CROWN). cProfile
revealed all of that was Phase 0 PGD attacks burning the time budget on
hopeless (UNSAT) cases:
  - `pgd_attack_general` (default 10 K restarts × 100 iters): 8.6 s
  - `_simple_pgd` (10 K × 50): 3.4 s
  - `onnx_torch_runner.pgd_via_onnx` (256 × 100): 0.8 s

Tuned to `pgd_restarts=128` + `pgd_time_budget_phase0=2.0` (in
`configs/cgan_2023.yaml`): every SAT case still caught (PGD finds the
witness in iter 0), and UNSAT cases reclaim **10 s back** per case.
Profile script: `scratch/cgan_pgd_sweep.py`.

## Algorithmic adds for this benchmark

These ops + bounds were added during the cgan run; reused by all future
generator-heavy benchmarks.

- **ConvTranspose** propagation (`zonotope.propagate_conv_transpose`,
  `network.py:ConvTransposeNode.zonotope_propagate`). Forward uses
  `F.conv_transpose2d` on center + each generator column. Backward
  CROWN uses `F.conv_transpose2d` (the adjoint of conv = conv_transpose
  and vice versa). Batched zono forward in `verify_zono_bnb` re-uses
  the same dispatch. ONNX-loader BN folding extended to also fold into
  ConvTranspose (`_fold_batchnorm`, axis=1 broadcast for C_out).

- **Sigmoid / Tanh closed-form linear bounds**
  (`verify_zono_bnb._sigmoid_tanh_linear_bounds`). NOT sampling —
  see CLAUDE.md operational rule. Cases:
  - Pure-convex / pure-concave (entire interval is on one side of the
    inflection point): the two-point secant + tangent at the endpoint.
  - Mixed (interval straddles the inflection): binary-search Newton
    for the tangent point `p` solving
    `σ'(p)·(hi − p) + σ(p) = σ(hi)` (lower bound when convex side has
    width > concave) or its mirror. Falls back to constant `y = σ(hi)`
    or `y = σ(lo)` when no tangent root exists in the interval.
  Validated for soundness across 200 random intervals × 1000 samples:
  zero violations (`tests/test_sigmoid_tanh_bounds.py`). Plumbed
  through gpu_graph + batched zono forward + backward CROWN.

- **Upsample-9 (nearest-neighbour)** as a linear-exact op. Forward:
  `F.interpolate(mode='nearest')`. CROWN backward: adjoint is
  `avg_pool2d(..., divisor_override=1)` over the same scale factor.
  Pure linear → no bound loss.

- **MaxPool / AveragePool / scalar Mul / Softmax / bilinear MatMul**
  bounds in the batched zono forward + CROWN backward
  (`verify_zono_bnb._forward_zonotope_graph_batched` and
  `_spec_backward_graph_batched`). AveragePool + scalar Mul are
  linear (no bound loss). MaxPool, Softmax, and bilinear MatMul use
  box relaxation: forward computes interval bounds (per-cell lo/hi),
  drops generator correlation, and emits a single new gen column per
  non-zero-slack cell; backward CROWN passes constants (slope-0) to
  `acc` (no signal to inputs). Sound but loose; sufficient because
  the only attention-bearing instances in cgan are SAT (caught by
  PGD before bounds matter). Softmax uses auto_LiRPA's
  `interval_propagate` formula (`docs/cgan_2023.md`). Tested in
  `tests/test_cgan_op_bounds.py` (10 cases, all soundness-verified
  on 300 random samples).

- **`onnx_torch_runner` raw-ONNX PGD fallback**
  (`src/vibecheck/onnx_torch_runner.py` →
  `verify_graph._input_split_batched` exception path). Last resort
  when gpu_graph build/forward fails on a model whose ops aren't in
  the bound pipeline. Used to bootstrap small_transformer SAT
  detection before MaxPool/Softmax/bilinear-MatMul were added; kept
  as a safety net.

- **onnxsim auto-preprocessing** for attention-shaped subgraphs
  (`onnx_loader.load_onnx` heuristic: any model with Softmax +
  ≥5 MatMul nodes). Folds the 68 MatMul + 44 Div constant-spectral-
  norm scaffolding in small_transformer that our loader otherwise
  can't ingest. Skipped on FC-only nets because onnxsim's
  MatMul+Add→Gemm fusion measurably loosens milp_verify bounds on
  acasxu prop_3.

## Knobs (`configs/cgan_2023.yaml`)

- `input_split_batched_enabled: true` + `input_split_batched_clip_enabled:
  true` — same proven combination from cersyve/cifar100/tinyimagenet.
- `input_split_batch_size: 16` — kept small because the imgSz64
  generators allocate ~600 MB per forward at batch=16 already; larger
  batches OOM on the 10 GB GPU. imgSz32 nets would tolerate 64.
- `input_split_batched_alpha_iters: 0` and `_alpha_boundary_eps: 0.0`
  — α-opt at the input-split level didn't measurably improve any
  case; we lean on the per-leaf zono+CROWN tightness instead.
- `input_split_batched_milp_escalate: false` — MILP escalation
  doesn't help on these conv generators; the zono pipeline alone
  closes them within budget.

## Pre-merge bug audit (uncovered during this benchmark)

- **Multi-disjunct DNF clipping was SOUNDNESS-broken**
  (`_clip_box_by_halfspaces_batched`): the per-query halfspaces were
  intersected naively as an AND, but the spec is a DNF (∨ of
  conjuncts). Naive AND can prove a leaf empty when the actual DNF
  union is non-empty → spurious `verified` on multi-disjunct SAT
  cases. Fixed via per-disjunct clipping followed by union-bbox.
- **`pgd_attack_general` only reduced loss within a single disjunct**
  (the first). Multi-disjunct SAT cases that need to be witnessed
  via disjunct ≥1 were silently missed. Added `_simple_pgd` fallback
  that takes the min-over-disjuncts as the loss → SAT cases
  reachable from any disjunct are now caught.
- **OOM-halving floor (64 restarts) was too high** for imgSz64.
  Changed to 1, so OOM-halving naturally trails off rather than
  abandoning the case.

## Known unsolved cases

None at 120 s/case. Open follow-ups (not required for completion):

- **Tighter MatMul / Softmax bounds via McCormick relaxation**. The
  current bilinear MatMul uses a naive 4-corner box (sound but quite
  loose); auto_LiRPA uses McCormick envelopes for variable-times-
  variable products. Would close UNSAT cases on transformer-bearing
  benchmarks (e.g. vit_2023, ml4acopf). Not needed for cgan since
  both small_transformer cases are SAT and caught by PGD.
- **α-optimization for sigmoid/tanh tangent points**. The current
  binary-search Newton gives a fixed tangent; auto_LiRPA does an
  α-opt over the tangent location. Could close some marginal UNSAT
  cases faster (not needed for 21/21).
- **acasxu-style `verify_hybrid` for cgan**: tried, doesn't apply.
  verify_hybrid grabs `n_output` from the last `fc` op and uses an
  FC-tail-specific spec backward. cgan ends in Conv, not FC. Would
  need a Conv-tail variant; `input_split_batched` already wins on
  cgan so the work isn't justified. See `scratch/cgan_hybrid_compare.py`.
