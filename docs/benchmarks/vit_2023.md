# vit_2023 (extended track)

Two Vision-Transformer nets, 100 eps-robustness instances each, 100 s
per-instance budgets (`instances.csv`):

- `ibp_3_3_8.onnx` — 3 attention blocks, ~110 ops post-fold;
- `pgd_2_3_16.onnx` — 2 attention blocks, wider per-token dim.

Each spec is 9 single-constraint disjuncts (targeted misclassification).
**The instance set has no SAT cases** (ABC published: 0 sat) — attack
cannot score; everything comes from bounds. The nets exercise the full
attention pipeline: `MatMul(Q,Kᵀ) → softmax → MatMul(attn,V)`, i.e.
bilinear products of two perturbed operands plus softmax.

| pgd_2_3_16 | **47/100 unsat** (with-PGD sweep); the no-PGD soundness sweep with the final code closed **59/100** — final production sweep running, pending confirmation | 60/100 | all confirmed solves mutual with ABC |

Total: **119 confirmed** (up to 131 pending the final pgd sweep) vs ABC's
published **125** (65 + 60). 0 sat verdicts anywhere on either tool.

Score history (what moved the needle): ibp 57 (first sweep) → 59
(kfsb-lite + refine rounds) → 72 (max-shift + joint intermediate-alpha);
pgd 2 → 45 → 47/59 (same stack). Mutual ibp solves run ~2× faster than
ABC (e.g. `ibp_3_3_8_2759`: 23.5 s vs ABC's 50.6 s); on the shared
grind case `ibp_1028` our root alpha bound is tighter than ABC's
(−0.0393 vs −0.0438) at ~3× their domain throughput (live ABC run, same
box).

## Algorithmic wins (vs published ABC reference)

All mechanisms live in `attn_crown.py` + the softmax decomposition in
`network.py`; the chain is wired at the end of `verify_graph.py`
(refine → joint alpha → per-query backward alpha → beta-BaB →
relu-split). Full campaign log: `scratch/VIT_FINDINGS.md`.

1. **Softmax decomposition with exact max-shift.** Softmax is emitted as
   the primitive subgraph `exp → reduce_sum → reciprocal → mul_bilinear`
   (canonical-ops move — every downstream chain gets it for free), with
   ABC's exact max-shift: `exp(x − x_k)` where `k` = concrete center
   argmax treated as a fixed-index gather (exact linear by softmax
   shift-invariance; `retarget_softmax_shifts`). The shift is what makes
   the pgd family tractable — its raw exp inputs span ~[−10, 0] after
   shifting instead of blowing up the chord relaxation. Backward CROWN
   handlers for the primitives (exp tangent/chord, reciprocal
   chord/tangent, McCormick planes for `mul`/`matmul_bilinear` with the
   linear term distributed to both inputs, per-op input bounds recorded
   during the plain forward) reach plain-CROWN parity with ABC
   (ibp_1028: −0.056 = ABC; 2759: −0.034 vs ABC −0.039).

2. **Joint differentiable-intermediate alpha** (`attn_alpha_joint`).
   The ABC mechanism that closes their pgd solves without BaB: the spec
   loss backprops *into the planes of the intermediate-bound backwards*
   (±I batched walks re-derived inside the autograd graph each
   iteration), jointly over all open queries (joint C matrix). Ours
   adds per-row alphas on the capped ±I rows, straight-through-estimator
   gradients through the bound intersections, and a sign-jump
   (vertex-seeking) warm start. fp32 search + fp64 certification pass;
   certified bounds are intersected into the shared root state for every
   later phase. Measured: pgd_2043 root −1.31 → positive in a few steps
   (ABC needs 2); family effect pgd 2 → 45+, ibp 59 → 72.

3. **Batched no-reforward beta-CROWN BaB** (`attn_beta_bab`).
   ReLU splits AND bilinear-input (exp-input value) splits, each with
   beta Lagrangian multipliers — clamp-only splits provably stagnate
   (pinned in `tests/test_bilinear_split.py`: width-1e-6 value splits
   left the toy bound at the full-box value; the beta term is what
   restricts the concretization domain, same for relu splits via
   `rbeta`). Domains are bounded 32-at-a-time in one autograd graph with
   parent warm-starts and *no zonotope re-propagation* (root state
   intersected per domain — the ABC interm-fixed recipe). fp32 search
   with fp64 recheck of every closure (`attn_bab_fp32`; soundness never
   rests on fp32). kfsb-lite branching (top-4 cheap-eval FSB) closes
   cases plain ew-weighted branching missed. Throughput: 347 → ~7200
   domains/300 s (~20×) on ibp_1028.

4. **CROWN intermediate refinement** (`attn_refine_op_bounds`): batched
   ±I backward per relaxation-feeding node, tightening the exp/McCormick
   plane input ranges. One 0.7 s pass moved pgd_7086 roots −8.6 → −4.1
   (plain) / −7.7 → −3.05 (alpha); refine↔alpha rounds compound
   (pgd_7086 q3: −1.16 → +0.0009 → +0.23 over two rounds).

5. **Loader/emission groundwork** (zero new handler types): symbolic
   batch-dim pinned to 1 before onnxsim (29 surviving shape ops → 0),
   ReduceMean/Transpose/Concat/N-D MatMul/BatchNorm emitted as canonical
   fc/slice ops — several of the replaced legacy handlers were latently
   unsound (Transpose silent passthrough, Concat dropping const blocks).
   Point-prop vs onnxruntime: ibp 8.9e-7, pgd 1.4e-6.

## Knobs (`configs/vit_2023.yaml`)

- `phase2_crown_enabled: true` — Phase-2 backward CROWN carries the
  attention handlers (exp/recip planes, McCormick); it is the bound
  source, not the zono forward (ABC's plain backward is ~4-5× tighter
  than forward zono on these nets).
- `zono_lift_enabled: false`, `max_tighten_layer: 0`,
  `bab_refine_passes: 0`, `skip_phase8_milp: true`,
  `phase1_method: plain` — the generic FC/conv tightening machinery
  (Phase-2.5 LP lift, per-layer MILP tighten, bab_refine cascade,
  Phase-8 gen-LP/MILP) has no `mul_bilinear`/softmax-primitive support
  (those builders would raise `NotImplementedError`), and the binding
  slack is the bilinear/softmax remainder, not ReLU triangle slack
  (relu-split-only BnB closed nothing in 513 nodes pre-backward).
  Plain Phase 1 records the per-op input bounds the backward planes
  need.
- `parallel_pgd_enabled: false`, `pgd_restarts: 128` — no SAT cases
  exist, attack can't score; keep a modest PGD only as a counterexample
  sanity check. (The with-PGD vs no-PGD pgd-family delta — 47 vs 59 —
  is exactly this overhead eating bound time; final production sweep
  pending.)
- `zono_input_split_enabled: true` + `zono_input_split_max_nodes: 0` —
  the flag gates the whole last-chance attention chain in
  `verify_graph.py` (root refine → joint alpha → per-query bw-alpha →
  beta-BaB → relu-split); the cap of 0 disables the literal *input*-split
  fallback itself (useless on image inputs).
- `zono_relu_split_max_nodes: 6000` — node cap for the relu-split
  fallback that runs after the beta-BaB.
- `attn_bab_batch: 32`, `attn_bab_iters: 12` — the ABC vit recipe
  (batch 32 domains/call, ~10 beta iters); batching is the 20×
  throughput lever.
- `attn_refine_passes: 2` — root intermediate refinement; the big pgd
  lever (−7.7 → −3.0 in 0.7 s), marginal on ibp (intermediates already
  tight there).
- `attn_refine_rounds: 1` — per-query refine↔alpha iteration; rounds
  compound but each costs wall, 1 round leaves the BaB its share.
- `attn_alpha_joint: true` — the joint differentiable-intermediate alpha
  (win #2). `attn_joint_max_rows` (default 4096) caps the ±I autograd
  rows — required on the 3-block ibp net (~20k rows OOM'd the A10G).
- `attn_joint_max_s: 45.0`, `attn_joint_frac: 0.5`,
  `attn_joint_iters: 80` — joint-phase wall budget and iterations. The
  pgd-family deep-gap closes need ~50 alpha iterations (ABC's own
  ablation: `iteration: 2` flips their pgd unsat → timeout), while
  shallow-gap (ibp-style) queries are budget-gated by gap depth
  (`attn_joint_deep_gap`/`attn_joint_s_per_q` defaults) so the BaB keeps
  the wall — alpha overhead measurably cost a borderline ibp case (6769)
  before the gate.

## Reproduction

```bash
# single case (BENCH = vnncomp2025_benchmarks/benchmarks)
.venv/bin/python -m vibecheck.main \
  $BENCH/vit_2023/onnx/pgd_2_3_16.onnx \
  $BENCH/vit_2023/vnnlib/pgd_2_3_16_2043.vnnlib \
  --config configs/vit_2023.yaml --timeout 100 --results-file /tmp/r.txt
cat /tmp/r.txt   # verdicts come from the results file, never exit code

# integration pins
.venv/bin/python -m pytest tests/integration/test_vit_2023.py -m integration
```

Full sweeps run on the AWS g5 (A10G, 24 GB — the joint alpha OOMs
smaller cards at default row caps): loop `instances.csv` with
`--timeout 100 --results-file`, counting verdicts from the file only.
Sweep results archived under `~/persistent_runs/vit_sweep_{ibp,pgd}_*`
on the AWS box (e.g. `vit_sweep_ibp_0611_1639` for the 72/100 ibp run).

## Integration tests (`tests/integration/test_vit_2023.py`)

No SAT cases exist, so all three pins are UNSAT:

- `pgd_2_3_16_2043` (~12 s) — the joint-alpha happy path: ABC's easiest
  pgd solve, closed at the root by max-shift + joint
  intermediate-alpha with no BaB. Regression guard for wins #1/#2.
- `ibp_3_3_8_2759` (~21 s) — the BaB path on the 3-block net: root alpha
  leaves a gap, beta-BaB closes in ~39 domains. First vit case ever
  closed; guards the backward handlers + beta-BaB end-to-end.
- `pgd_2_3_16_8836` (~99 s) — deep-gap alpha close: exercises the
  long-iteration joint-alpha regime (`attn_joint_iters`) plus
  refine rounds. `max_wall_s` 150 (~1.5× observed).

## Known unsolved

- **ibp: 28 open** — 8 misses where ABC solves slowly (52–106 s of
  their budget; our roots plateau at −0.007..−0.037 around ~4k domains)
  plus ~20 both-timeout.
- **pgd: remaining misses** vs ABC's 60 (final count pending the
  production resweep; 59/100 on the no-PGD sweep with final code) —
  ABC's extra solves are its 39–50 s BaB-stage closes.

Both families' open cases need the same levers: further
alpha-convergence (better joint-alpha optimization of the deep-gap
roots) and/or BaB improvements — not new relaxation machinery; the
roots are within BaB range but the trees don't close in 100 s.

**Soundness**: the no-PGD soundness sweep is clean — 0 sat verdicts
anywhere (the benchmark has none, so any sat would be a false
positive). Every bounding mechanism is Lagrangian/intersection-based,
fp32 closures are fp64-recertified, and the campaign added test pins
for the split/beta/joint mechanics (`tests/test_bilinear_split.py`,
`tests/test_attn_batch.py`, `tests/test_invprop_tighten.py` among
them).
